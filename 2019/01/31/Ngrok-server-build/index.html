内网穿透之 Ngrok 自搭建

字数统计: 694 | 阅读时长 ≈ 2

最近遇到一个问题，我们都知道，公司或者实验室的服务器一般均是位于内网的，在外无法直接访问。对于公司或者校园网，一般都会有提供 VPN 进行登陆访问，而所在实验室并没有，那么如何才能让我们人没在也可以对内网的服务器进行访问？ Ngrok 就是很棒的一个解决方案。

Ngrok 简介

Ngrok 是什么？从它官网的描述来看，“Ngrok exposes local servers behind NATs and firewalls to the public internet over secure tunnels.” 所说的是 Ngrok 通过安全的通道将在 NATS 和 防火墙之后的本地服务器暴露到公共网上，这就是我们想要的结果。也称为内网穿透。

Ngork 工作流程如下：

  • 本地内网主机 A 和公网服务器 B 建立一条连接
  • 用户访问服务器 B
  • 服务器 B 联系内网主机 A 获取内容
  • 服务器 B 再将获取的内容发送给用户

其实，我们可以看的出，服务器 B 就好比是一个交换的公网路由器。

Ngrok 官网有推出免费云服务，仅需下载客户端运行即可完成，但是其服务器在国外且可能被墙，所以我们最好自己搭建一个 Ngrok 服务。

要搭建服务，我们需要具有以下资源：

  • 一台具有公网 IP 的服务器
  • 一个域名 （可选）
